Analyzing usage of Citibikes in New York City for January 2019¶

This notebook is using data for the usage of citibikes in New York City in January 2019. Pulled from this website, here is the data I'm using to analyze the following:

  • How long do people usually ride rental bikes for, and does this have correlation with the distance traveled?
  • How is each bike being used?
  • What is the main demographic of rental bike users in NYC?

According to the website that I pulled the data from, here is the schema defined on this csv file:

  • Trip Duration (seconds)
  • Start Time and Date
  • Stop Time and Date
  • Start Station Name
  • End Station Name
  • Station ID
  • Station Lat/Long
  • Bike ID
  • User Type (Customer = 24-hour pass or 3-day pass user; Subscriber = Annual Member)
  • Gender (Zero=unknown; 1=male; 2=female)
  • Year of Birth

"This data has been processed to remove trips that are taken by staff as they service and inspect the system, trips that are taken to/from any of our test stations (which we were using more in June and July 2013), and any trips that were below 60 seconds in length (potentially false starts or users trying to re-dock a bike to ensure it's secure)."

One interesting thing to note about this is that even if there is a not null value for the "Gender" column, any value with 0 is "unknown," which may make it harder to give us a real insight without filtering all 0 values out.

Trip time accuracy¶

One thing that I was curious about is what the tripduration field represented. The citibike website did not make it clear what this field meant. We're the bikes smart enough to track how long the bikes were actually used? Did it not count the time the bike was stationary? Or is it just the total length the bike was rented out for? To figure this out, I did my own calculation. I did this by generating a timedelta object from the difference in start and stop times, then converting them to minues.

In [10]:
trip_seconds = []
trip_time = []

for index in range(len(citibikedata)): 
    elapsed_time = citibikedata["stoptime"].iat[index] - citibikedata["starttime"].iat[index]
    elapsed_seconds = elapsed_time.total_seconds()
    elapsed_minutes = elapsed_seconds / 60
    trip_seconds.append(elapsed_seconds)
    trip_time.append(elapsed_minutes)
    
#I'm actually adding these calculated values to the dataframe because it's nicer to work with minutes 
citibikedata = citibikedata.assign(trip_time=trip_time)

Now we can compare my calculated field (which just represents how long the bike was rented out for) and the tripdata field

Analyzing the data¶

Geolocation data¶

Since I want to determine average trip distances, I am going to calculate it and append it to the dataset. To do this, I'm going to calculate the distance traveled from the start to the finish using the python GeoPy package. I didn't just use a simple distance formula, since the earth is curved and calculating distances accurately involves some more complex computation that GeoPy takes care of for me. While I belive this is negligible at city scale, I decided to still use it to make sure it's as accurate as posible.

In [13]:
distance_traveled = []

for row in range(len(citibikedata)): 
    coord1 = (citibikedata["start station latitude"].iat[row], citibikedata["start station longitude"].iat[row])
    coord2 = (citibikedata["end station latitude"].iat[row], citibikedata["end station longitude"].iat[row])
    distance_traveled.append(geopy.distance.distance(coord1, coord2).km)

First, I want to transform the data by calculating the actual age, since birth year is a relatively useless data point. I binned the ages, since I wasn't interested to know the exact ages, but it would be easier to make inferences off of binned data:

In [30]:
binned_ages = []

for index in range(len(citibikedata)): 
    if (2019-citibikedata["birth year"].iat[index] < 11):
        binned_ages.append("0-10")
    elif (2019-citibikedata["birth year"].iat[index] < 21):
        binned_ages.append("10-20")
    elif (2019-citibikedata["birth year"].iat[index] < 31):
        binned_ages.append("20-30")
    elif (2019-citibikedata["birth year"].iat[index] < 41):
        binned_ages.append("30-40")
    elif (2019-citibikedata["birth year"].iat[index] < 51):
        binned_ages.append("40-50")
    elif (2019-citibikedata["birth year"].iat[index] < 61):
        binned_ages.append("60-70")
    else:
        binned_ages.append("70+")
        
citibikedata = citibikedata.assign(age=binned_ages)

While I wasn't surprised that older people tend to use the bikes less, and teenagers and children didn't use citibike (less incentive to spend money on rental service), I was surprised that there were more 30-40 year olds than there were 20-30 year olds. I would have guessed that young professionals rented more bikes, but it seems that the service is the most popular among 30-40 year olds.
